## 2. Pre-decided (from the spec — not relitigated)

- **14-day trial**, DPAPI-encrypted `license.dat` at `%APPDATA%\QuickSay\` (survives uninstall → trial-reset resistance, spec §5.6/§5.7).
- **Blocking paywall** after expiry (spec §11 Q1).
- **Pricing NOT hardcoded** — `$39` launch (first 500 orders) → `$74` regular + financing, flipped **server-side** at 500 orders. The app fetches price **and** checkout URL from `GET /pricing`.
- **Ed25519-signed JWT**, 14-day `exp`, 7-day grace, then `RE_VALIDATION_NEEDED` (spec §3.2).
- **LemonSqueezy** processor; checkout/product URL assigned at M.3.

## 3. Decisions made this session (the Phase-4 micro-decisions)

| # | Decision | Choice | Rationale |
|---|---|---|---|
| **U1** | Paywall dismissibility | **Closeable window, but recording stays gated.** Closing the modal does not enable recording; the next hotkey press re-shows it. | Honors spec §11 Q1 ("cannot dismiss into a working recording state") **without trapping** the user in an unclosable window — QuickSay is a background tool, not a kiosk; the user may need other apps. The gate lives on *recording*, not on the window. |
| **U2** | Trial countdown banner | **From day 11 of 14 (≤4 days remaining), once per launch**, via a gentle TrayTip + tray tooltip ("trial: N days left"). | The session default. Early enough to prompt action, late enough not to nag from day 1. Tray tooltip always reflects state for at-a-glance awareness. |
| **U3** | "Buy now" visibility | **Always available** — tray "🔑 License / Unlock…" item + a "License" tab in Settings (visible from day 1). During an active trial the tray item opens the paywall in `welcome` mode. | The session default; lets eager users purchase early without friction. |
| **U4** | Activation retries | **No artificial retry cap.** Each attempt surfaces a specific, actionable message; rate-limiting + the "contact support" affordance live server-side (Worker 429 + `support@quicksay.app` footer). | A hard client retry counter mostly frustrates honest users with typos. The Worker already rate-limits (`/activate` 10/hr, spec §2.5); the friendly 429 copy ("try again shortly") handles abuse. The support email is always one click away in the footer. (Deviation from the session's "3 retries" default — justified: the server is the real limiter; a client counter adds friction without security value.) |
| **U5** | Where activation lives | **Both** the paywall modal (tray process) **and** the Settings "License" tab. | The paywall is the primary, blocking path; the Settings tab is the calm, always-available path (and shows licensed status). Settings activation posts `0x5555` to the tray so the running engine lifts the gate immediately. |
| **U6** | Crypto on the hot path | **Verify once per token, cache in memory.** The Ed25519 verify (~1–2 s) runs on a deferred startup timer and on token change; the per-recording gate reuses the cached verdict and only re-derives the time-based state. | A multi-second verify per hotkey press is unacceptable. The cache is keyed on the exact JWT string, so a forged/changed token always re-verifies (no security loss — a local attacker cannot make a *different* token reuse a prior "valid" verdict). Off the hot path per spec §1. |

## 4. Paywall modal contents (as built)

- Headline (mode-driven): *"Your free trial has ended"* / *"Welcome to QuickSay"* / *"Your license needs attention"*.
- Subhead: *"One-time payment. No subscription. Lifetime updates."*
- **Price line** — populated at runtime from `GET /pricing`; renders `$39 USD · one-time, lifetime — Launch price: N left`, or `$74 … Pay-over-time options available at checkout`, or (offline) `Get lifetime access — see price at checkout`. **Never hardcoded.**
- Primary CTA **"Get my license"** → opens the LemonSqueezy `checkoutUrl` from `/pricing` (falls back to the `LEMONSQUEEZY_PRODUCT_URL` placeholder, then `quicksay.app/buy`).
- Secondary CTA **"I already purchased — enter my license key"** → reveals key input + Activate, with inline success/error.
- Footer: Terms · Refund policy · support@quicksay.app.

## 5. Notable engineering decisions / spec deviations (logged for review)

- **Pure-AHK Ed25519 verifier (no DLL).** Windows CNG/BCrypt has no Ed25519 provider, so `lib/ed25519.ahk` implements edwards25519 verification in AHK (base-2¹⁷ bignum, limb-aligned to 2²⁵⁵ for a cheap mod-p fold; RFC 8032 reference point arithmetic; SHA-512 via BCrypt). Verified against the **RFC 8032 §7.1 KAT** and **Node interop**. Keeps the verifier self-contained and baked into the binary (no bundled `libsodium.dll`), matching the spec's "verifies offline with a public key compiled into the binary."
- **HTTP in `license.ahk`, not `lib/http.ahk`.** Spec §2.7 says to reuse `HttpGet()`/`HttpPostFile()`, but `lib/http.ahk` only has a multipart file-upload helper (no JSON GET/POST) and the session forbids modifying it. License calls use a private WinHTTP COM helper inside `license.ahk` (same COM object the existing code uses), keeping `http.ahk` untouched and `license.ahk` decoupled/testable.
- **Dedicated license-dat mutex** (`Local\QuickSayLicenseDatLock`) rather than `AcquireConfigLock()`. `license.dat` is a separate file from `config.json`; a dedicated lock serialises its RMW without coupling `license.ahk` to QuickSay.ahk internals (so it unit-tests standalone).
- **AHK case-insensitivity gotcha** (fixed during the build): a local named `json` shadows the `JSON` class. Renamed to `plain`; noted in code.

**Live UI — verified in the real WebView2 host** (a harness called `PaywallUI.Show("expired")` — the exact code path used by the startup auto-show and the recording-gate trigger): the actual paywall window renders correctly (logo, "Your free trial has ended", subhead, dynamic price line from live `/pricing`, teal "Get my license", "I already purchased" link, Terms·Refund·support footer). **This step caught a real production bug the browser/unit tests missed** (they bypass the AHK GUI host): `PaywallUI.Show` set the window icon with AHK's `SendMessage(…, "ahk_id " hwnd)` *before* `gui.Show()`, which throws "Target window not found" with `DetectHiddenWindows` off — so the paywall would have failed to open in the real app. Fixed by using `DllCall("SendMessage", …)` on the raw HWND (the same pre-`Show` pattern settings-ui uses). Re-verified after the fix.

## 8. 60-second "first-run user" script (text)

- **Day 0 (install & first launch):** QuickSay's tray icon appears. In the background it starts a 14-day trial (writing an encrypted `license.dat`). The user holds Ctrl+Win, speaks, releases — text appears. No mention of payment. Tray tooltip: *"Idle — trial: 14 days left."*
- **Day 11:** On launch, a gentle TrayTip: *"Your free trial ends in 3 days. Unlock QuickSay anytime from the tray menu."* Dictation still works normally. The tray menu now nudges via *"🔑 License / Unlock…"* and Settings → License shows the countdown + price.
- **Day 14 (still trialing):** *"…ends in 1 day."* Everything still works.
- **Day 15 (expired):** On the first hotkey press, recording is refused and the **paywall** appears: *"Your free trial has ended,"* with *"$39 USD · one-time, lifetime — Launch price: N left,"* a **Get my license** button, and an **enter-my-key** option. The app stays open (settings/help work); only recording is gated.
- **Purchase:** The user clicks **Get my license** → the LemonSqueezy checkout opens in their browser (price shown there, financing surfaced on the $74 tier). They pay, receive a key by email, paste it into the paywall (or Settings → License), and click **Activate**. The app contacts the license worker, stores the signed license, dismisses the paywall, and dictation resumes instantly. Tray: *"QuickSay is now licensed. Thank you!"*
