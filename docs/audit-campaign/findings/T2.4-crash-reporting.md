## The contract shipped (from spec §6, no ambiguity — nothing needed asking)

- **DSN location:** compiled constant `SENTRY_DSN_PUBLIC` (public DSN only — never `sentry_secret`) in `lib/crash-reporter.ahk`. **Empty placeholder** → the reporter builds/sends nothing until baked in. **M.1 sets the staging/project DSN; M.3 confirms prod.**
- **Environment tag:** `SENTRY_ENVIRONMENT := "beta"` (M.3 flips to `production`).
- **Allowlist (the envelope carries ONLY these):** `release`, `environment`, `level`, `platform`, `exception.type` (scrubbed), `exception.value` (scrubbed), `tags.hotkey_mode`, `tags.last_action`, `contexts.os` (`name:"Windows"` + scrubbed generic build, never machine name), `extra.line_file` (scrubbed), `extra.line_number` (int), `extra.this_func` (scrubbed). Anything else (transcript, finalText, paths) is **not attached** — allowlist, not blocklist.
- **`last_action`** derived from existing audited signals (`isRecording`/`isProcessing`, spec §6.7) — no new hot-path instrumentation, no content.
- **Throttle:** in-memory rolling 60-min ring, max 5 sends; excess dropped + debug-logged.
- **Opt-in (default OFF):** hard no-op until `crashReportingEnabled=true` **AND** `crashReportingPrompted=true`. No envelope is ever sent before the user answers the modal.
- **Opt-in modal copy (verbatim, spec §6.5):** "Help us fix bugs? QuickSay can send anonymous crash reports — no transcripts, no audio, no personal information. [Yes, help out] / [No thanks]." (WebView2 modal expands the reassurance; MsgBox fallback uses the same plain-English consent question.)
- **Crash-safe:** the `OnError` handler is fully try-wrapped and returns `0`, so a bug in the reporter never masks the user's real error — AHK's default handling still proceeds.

## Security review (independent `comprehensive-review:security-auditor`, 2026-06-02)

The agent empirically ran the real scrubber against crafted inputs. The allowlist held everywhere **except one** field; fixes applied and re-verified:

- **P1 — `os_version` reached the wire without `Scrub()`** (the one allowlisted string field with no scrub; spec §6.4 says "never machine name"). **FIXED:** `contexts.os.version` is now `Scrub()`-wrapped. Added grep-gate coverage (`T6b`, and `wire-capture.ahk` now sets `os_version` to a machine-name-bearing value → 0 matches on the wire).
- **P1/P2 — JWT bypasses.** A 2-segment JWT (truncated signature) still carries the PII `email` claim, and a JWT wrapped at a dot boundary evaded the 3-segment rule. **FIXED:** rule 1 now matches the distinctive `eyJ…\.eyJ…` shape (header and payload both base64url(JSON) → both start `eyJ`), covering 2-or-3 segments and tolerating whitespace around the dots, with near-zero false positives. Tests `T2b`/`T2c`.
- **Accepted residuals (documented in code):** a `gsk_` key or JWT split by a newline **inside** a base64 segment is not stitched back (real key-echo errors are contiguous; a whitespace-tolerant key rule would greedily eat diagnostic text — a worse tradeoff; the allowlist (F12) is the load-bearing control and `exception.value` is a single error string, not a multi-line log). Username/machine backstops over-redact on common substrings — privacy-favoring, never a leak.

## Verification evidence

- **Unit suite:** 17 cases / 35 assertions, `tests/crash/run-tests.ps1` → **35 passed, 0 failed, exit 0** (offline; injected send-hook + clock/identity).
- **Gate 2 — PII grep on the real wire:** `wire-capture.ahk` builds the actual envelope bytes for a synthetic all-secrets error (fake key + fake JWT + user path + audio path + machine name in both message and `os_version` + transcript in `extra`). External grep → **0 matches** for `gsk_`, `eyJ`, the username, `.wav`, the computer name, the transcript, the audio stem.
- **Gate 3 — throttle:** 6 fires → 5 sent, 6th dropped + drop recorded; window roll resumes sending (`T8`–`T10`).
- **Gate 4 — opt-out:** `enabled=false` and `prompted=false` are both hard no-ops (`T11`/`T13`).
- **Syntax:** full `QuickSay.ahk` include tree compiles clean via Ahk2Exe.
- **Gate 1 — live Sentry event ≤10s:** **deferred to M.1** — requires a real `SENTRY_DSN_PUBLIC` (not yet provisioned; the constant ships empty). The envelope format is proven correct offline (valid 3-line JSON, accepted key order) and the POST path (`application/x-sentry-envelope`, `?sentry_key=…&sentry_version=7`, ≤5 s timeout) is wired and unit-covered via the send-hook.

1. **M.1:** set `SENTRY_DSN_PUBLIC` in `lib/crash-reporter.ahk` to the real project DSN; run gate 1 (trigger a deliberate error with reporting opted-in → confirm the event lands in the Sentry dashboard ≤10 s; capture the event ID).

3. Privacy policy: the report contents (allowlist) are non-PII by construction; no policy change needed beyond noting crash reporting is opt-in and anonymous.
