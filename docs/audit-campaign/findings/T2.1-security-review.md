# T2.1 — Security Review (STRIDE) of the T2 Production Systems Design Spec

**Session:** T2.1 (design gate) · **Date:** 2026-05-30 · **Method:** STRIDE threat model (`security-scanning:stride-analysis-patterns`) over `specs/T2-production-systems-design.md`, run inline by the design author with full trust-boundary context.
**Outcome:** **2 findings fixed in spec**, **6 accepted risks** (documented with rationale). No unmitigated high-severity threats remain.

## Trust boundaries & assets

- **App ↔ Worker** (HTTPS, untrusted network) — license keys, JWTs, machine_id cross here.
- **Worker ↔ LemonSqueezy** (server-to-server; LS API key) — never reaches the app.
- **LS ↔ Worker webhook** (HMAC-signed) — untrusted ingress until verified.
- **App ↔ Sentry** (HTTPS; public DSN) — scrubbed envelopes only.
- **App ↔ R2** (`version.json`, signed) — untrusted CDN content until verified.
- **At-rest:** `license.dat` (DPAPI, CurrentUser), CF secrets, KV (`sha256(license_key)` only).
- **Assets:** Ed25519 private key (critical), LS API key (critical), webhook secret (high), license JWTs (high), buyer email (PII), trial state (medium).

## STRIDE matrix

| ID | Cat | Threat | Verdict |
|---|---|---|---|
| S1 | Spoofing | Forge a JWT to fake a license | **Mitigated** — Ed25519; private key only in CF secret + offline backup; app verifies signature (§3.4 step 3) *before* trusting any claim. Forgery needs the private key. |
| S2 | Spoofing | Copy an activated `license.dat` to another PC | **Mitigated** — `machine` claim must equal local `ComputeMachineId()` (§3.4 step 5) → mismatch = LICENSE_REVOKED. (Clone caveat → A3.) |
| S3 | Spoofing | MITM/spoof the Worker to mint a fake license | **Mitigated** — TLS + a spoofed JWT fails the bundled-key signature check. (Can only deny, not grant → D4.) |
| S4 | Spoofing | Activate with someone else's license key | **Accepted by design** — license key = bearer identity (research §4); bounded by LS `activation_limit=1`. |
| S5 | Spoofing | Forge a LemonSqueezy webhook | **Mitigated** — HMAC-SHA256 over raw body, constant-time, verify-before-parse (§4.1). |
| T1 | Tampering | Edit `license.dat` to extend/reset trial | **Mitigated/Accepted** — DPAPI; injected JWT fails signature; future `trialStartedAt`→clock-rollback guard; reset→`TRIAL_BLOCKLIST`. Residual → A1. |
| T2 | Tampering | Swap `version.json` / installer (MITM, R2 compromise) | **Mitigated** — Ed25519 over canonical payload (§7.2), fail-closed verify (§7.3); `installer_sha256` commits to the binary; Azure code-sign is the 2nd layer. |
| T3 | Tampering | Alter the `/activate` JWT in transit | **Mitigated** — TLS + signature check. |
| T4 | Tampering | Flip `LICENSE_CACHE` `disabled`→`active` | **Mitigated** — only the HMAC-verified webhook writes KV; direct KV write needs CF-account compromise (= private-key trust level). |
| **R1** | **Repudiation/Replay** | **Replay an old `license_key_created` to re-grant a refunded license** | **FIXED IN SPEC** — §4.3 hardened to timestamp-monotonic + sticky-disable (an older/replayed event is a no-op; `disabled` is terminal). Was the one real gap. |
| R2 | Replay | Replay `/activate` | **Mitigated** — idempotent at LS + rate-limited (§2.5). |
| R3 | Repudiation | User denies activating | **Accepted** — LS holds the order record; Worker logs (scrubbed). Low relevance for a one-time license. |
| R4 | Replay | Reuse an expired JWT | **Mitigated** — `exp`/grace (§3.2); signature blocks extension. |
| I1 | Info disclosure | Log/transmit JWT, license key, API key | **FIXED/Mitigated** — §8.4 logging discipline added (normative); crash scrub list (§6.4); KV stores only `sha256(key)`. |
| I2 | Info disclosure | Transcript text in crash reports | **Mitigated** — allowlist (attach-only-safe) + "transcript never attached" (§6.4). (Debug-log transcript plaintext = core-engine T1.1-011 → A6.) |
| I3 | Info disclosure | Sentry DSN-with-auth leak | **Mitigated** — only the public DSN ships; never `sentry_secret` (§6.2, §8.2). |
| I4 | Info disclosure | Email PII in the JWT | **Accepted** — DPAPI at rest, TLS in transit, never logged; it's the user's own email (§3.1). |
| I5 | Info disclosure | Worker logs machine_id/email | **Mitigated** — §8.4 forbids it. |
| I6 | Info disclosure | Update-reject reason leaks to attacker | **Mitigated** — generic user message; detail only in local debug log (§7.3). |
| I7 | Info disclosure | KV dump = license-key dump | **Mitigated** — KV keyed by `sha256(license_key)` (§2.1/§2.2). |
| D1 | DoS | `/activate` flood | **Mitigated** — 10/hr per machine_id + CF WAF backstop (§2.5). machine_id rotation → A4. |
| D2 | DoS | Sentry quota flood via extracted DSN | **Accepted** — A2 (bounded to crash-visibility loss; v2.x forwarder is the lever). |
| D3 | DoS | KV read amplification on unmetered `/validate`,`/pricing` | **Accepted** — A4 (CF WAF + `/pricing` 60 s cache). |
| D4 | DoS | Worker/LS outage locks out paying users | **Mitigated** — 7-day grace + cached-`active` `/refresh` survives a 14-day LS outage (§2.4, §3.2; MASTER-PLAN §7). |
| E1 | Elevation | LS API key exposed to the client | **Mitigated** — key only in `src/lemonsqueezy.ts`; never in a response/app (§2, §8.2). The reason the Worker exists. |
| E2 | Elevation | Ed25519 private-key compromise | **Mitigated/Accepted** — CF secret + offline backup, never in git; rotation via `keyId` map (§7.4). Residual → A5. |
| E3 | Elevation | Trial→Licensed without paying | **Mitigated** — needs a Worker-minted JWT (LS-confirmed); injected token fails signature. |
| E4 | Elevation | Unverified webhook → arbitrary KV write | **Mitigated** — HMAC verify-before-parse gates all KV mutation (§4.1). |

## Fixes applied to the spec this session

1. **R1 — §4.3 rewritten** to *timestamp-monotonic + sticky-disable*: events apply only if their LS timestamp ≥ stored `updatedAt`; `disabled` (refund/delete) is terminal and cannot be flipped back to `active` by a stale/replayed `created`/`updated`. Closes the replay-to-re-grant vector.
2. **I1/I5 — §8.4 "Logging discipline" added** (normative, both sides): Worker never logs JWT / raw key / unhashed machine_id / email / secrets; app never logs JWT / key / Groq key. Promotes T2.2's prompt-only rule to a spec contract.

## Accepted risks (documented, not fixed)

| # | Risk | Rationale |
|---|---|---|
| A1 | Trial is tamper-*evident*, not tamper-*proof* (a user defeating both the `%APPDATA%` file survival **and** the server `TRIAL_BLOCKLIST` — e.g. a genuinely new machine — gets a fresh trial) | Acceptable at a sub-$100 price point; pirates aren't the target market (research §4; spec §5.5). Defense-in-depth (DPAPI + clock guard + survives-uninstall location + server blocklist) raises the bar without false-positiving real users. |
| A2 | Public Sentry DSN in the binary can be extracted to flood Sentry's quota | Impact bounded to loss of crash visibility (no data breach); Sentry server-side rate limits + inbound filters apply; the v2.x self-hosted forwarder (§6.1) is the documented lever if abused. |
| A3 | Identical `MAC + Windows ProductID` clones share a `machine_id` | By design — no VM detection / deeper HW fingerprinting (§5.5); avoids false-positives on legit virtualized/accessibility users. |
| A4 | machine_id rotation / floods on unmetered endpoints | Backstopped by Cloudflare WAF + per-IP limits, not app logic (§2.5); appropriate layer for volumetric defense. |
| A5 | Ed25519 private-key compromise is catastrophic (forge JWTs + updates) | Mitigated by storage discipline (CF secret + offline backup, never in git) and the `keyId`-map rotation procedure (§7.4); tracked in MASTER-PLAN §7 risk register. |
| A6 | Cross-ref, out of T2 scope: `debug_log.txt` logs transcripts in plaintext (T1.1-011); standard-user config write may fail under `{app}=Program Files` | Core-engine / installer concerns, not T2 systems. Flagged for the owning sessions; `license.dat` in `%APPDATA%` already sidesteps the Program-Files write issue. |

## Residual posture

No Spoofing/Tampering/Elevation threat is unmitigated. The one Replay gap (R1) is fixed in-spec. Information-disclosure is contained by allowlist + scrub + hashed-KV + logging discipline. DoS residuals (A2/A4) are volumetric, accepted, and backstopped by Cloudflare/Sentry platform controls.

## Independent security-auditor pass (2026-05-31)

After the inline STRIDE self-review above, an **independent `comprehensive-review:security-auditor` agent** reviewed the draft (the Phase-5 step the session names). It confirmed the core crypto/trust-boundary design and surfaced **4 P1 + 5 P2 + 3 P3 + 3 INFO** findings the self-review missed or under-rated. **All P1 and P2 are now fixed in-spec.**

| ID | Sev | Finding | Resolution (spec section) |
|---|---|---|---|
| **F1** | P1 | JWT `alg` header never validated → `alg:none` / algorithm-confusion surface | **Fixed** — §3.4 step 2 pins `alg=="EdDSA"`/`typ=="JWT"`; verifier hardcodes Ed25519, never reads alg from token; `version.json` has no alg field (§7.3). Decision D15. |
| **F2** | P1 | Server `TRIAL_BLOCKLIST` is architecturally inert (only read at purchase, where it must not act) | **Fixed** — user chose the **functional fail-open gate** (D17, 2026-05-31): §5.4 adds a best-effort `GET /trial/status` at trial start (fail-open when offline, re-checked once online), populated by `/trial/report`; gates trial start only, never a paid `/activate`. |
| **F3** | P1 | `/trial/report` unauthenticated, unrate-limited, permanent-TTL KV write → griefing/KV-pollution | **Fixed** — §2.5 rate-limits `/trial/status` (5/hr) + `/trial/report` (2/hr) + per-IP WAF; §2.2 TTL ~18 mo; the gate is fail-open and never blocks a purchase, so a poisoned/sprayed entry can at worst deny a *free trial* to a machine whose `SHA256(MAC+ProductID)` the attacker already knows (residual **A7**). |
| **F4** | P1 | Refund's own trailing `license_key_updated` (strictly-newer ts) could merge `status:active` back | **Fixed** — §4.2/§4.3: `updated` never touches `status`; disable is terminal; re-grant only via `created` with a new `orderId`. Decision D16. |
| **F5** | P2 | `version.json` canonicalization escaping under-specified → signer/verifier byte-mismatch | **Fixed** — §7.2 pins minimal RFC 8259 escaping, no `/`-escape, no `\u` of non-ASCII; T2.5 KAT must include a non-ASCII changelog entry. Decision D20. |
| **F6** | P2 | `installer_sha256` signed but not verified today → integrity binding decorative | **Fixed (re-rated)** — §7.3 marks it "mitigated-pending"; binary integrity = Azure code-sign + TLS until app downloads+hash-checks (verify-before-exec, fail-closed). T2 verdict downgraded. Decision D21. |
| **F7** | P2 | Grace-table intervals overlap; leeway applied asymmetrically | **Fixed** — §3.2 uses `effectiveExp` half-open partition; leeway once at the access cutoff. Decision D22. |
| **F8** | P2 | machine_id-keyed rate limits defeatable; `/activate` LS-amplification + key-validity oracle | **Fixed** — §2.5 adds per-IP WAF + global `/activate` ceiling; §2.3 collapses `not_found`/`disabled`→`invalid`. Decisions D18/D19. |
| **F9** | P2 | Webhook with missing/unparseable timestamp → monotonic-gate undefined | **Fixed** — §4.3 names the timestamp field; absent/unparseable → reject(400), never default; dedup before gate. Decision D16. |
| F10 | P3 | `instanceId:null` refund → no-op LS deactivate could 500-loop | **Fixed** — §4.2 skips the LS call, still disables, returns 200. |
| F11 | P3 | `license.dat` surviving uninstall = PII footprint | **Fixed** — §5.7 adds a privacy disclosure + "remove my data" affordance requirement for M.3. |
| F12 | P3 | Scrub regex fragile; only allowlist load-bearing | **Fixed** — §6.4 states allowlist is the control + broadens username pattern (any drive + UNC). |
| I-a | INFO | Local private-key deletion is the likely compromise path | **Incorporated** — §8.1 makes deletion a MANDATORY M.1 gate. |
| I-b | INFO | `/deactivate` unrate-limited (replay-grief) | **Incorporated** — §2.5 adds it to the per-IP WAF set. |
| I-c | INFO | Bake-in could ship a wrong trust anchor | **Incorporated** — §8.1 requires a build-time `publicKeySha256` assert in T2.3/T2.5. |

**Independent verdict:** the spec was "conditionally safe to freeze" pending the 4 P1 fixes; **all 4 P1 + all 5 P2 are now resolved in-spec**, so the boundary is safe to freeze on approval. The one user-facing posture choice, **D17** (trial blocklist), was resolved with the user on 2026-05-31 → **functional fail-open gate**.

## Residual posture (final)

All P1/P2 from both the self-review and the independent pass are fixed in-spec. Remaining accepted risks: A1 (tamper-evident trial), A2 (DSN-in-binary quota abuse), A3 (MAC+ProductID clones), A4 (volumetric → CF/Sentry controls), A5 (private-key-loss criticality), A6 (out-of-scope core-engine/installer cross-refs), **A7 (trial-blocklist victim-grief — bounded: requires the victim's `SHA256(MAC+ProductID)`, rate-limited, fail-open, and never blocks a purchase)**. The spec is approved by the user (2026-05-31) and frozen as the T2.2–T2.5 interface boundary.
