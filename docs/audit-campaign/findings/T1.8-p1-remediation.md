## The five P1s — before / after

### P1 #1 — T1.3-023: user data lived under `{app}`, not `%APPDATA%\QuickSay\` (HEADLINE)

**Before.** Every user-data path resolved from `A_ScriptDir` (`QuickSay.ahk` `ConfigFile`/`HistoryFile`/`StatsFile`/`DictionaryFile`/`AudioDir`, `lib/settings-ui.ahk` statics, `onboarding_ui.ahk` config + marker, ~53 `debug_log.txt` sites). Installed, `A_ScriptDir = {app} = %LOCALAPPDATA%\Programs\QuickSay Beta\`, so config/history/stats/audio/logs all lived in the program dir. Meanwhile T2.3 placed `license.dat` (which also holds **trial** state: `trialStartedAt`, `trialMachineId`) under `%APPDATA%\QuickSay\`. **The two halves disagreed**, and the uninstall "don't keep data" path `DelTree {app}\data` would have wiped a trial file had it lived under `{app}` — trivially resetting a trial by reinstall.

**Decision: migrate user data to `%APPDATA%\QuickSay\`** (matches T2.3 + the `quicksay-go-to-paid` skill). New shared resolver **`lib/datadir.ahk`**:

- `GetDataDir()` → `%APPDATA%\QuickSay` when `A_IsCompiled` (production), else `A_ScriptDir` (dev — keeps the test harnesses + live runs using `Development\`). Depends only on built-ins, so it is safe to call from a class static initializer regardless of `#Include` order.
- Routed through it: `config.json`, `dictionary.json`, `data\history.json`, `data\statistics.json`, `data\audio\`, `data\logs\` (via `GetDebugLogPath()` — a function so the ~53 debug sites need no per-function `global` decl), and the `data\onboarding_done` marker. **Shared by all three processes** (tray `QuickSay.ahk`, settings `settings_ui.ahk`/`lib/settings-ui.ahk`, wizard `onboarding_ui.ahk`) so they can never split-brain on where config lives.
- **Bundled, read-only assets stay install-relative**: `sounds\`, `gui\`, exes, ffmpeg, the AutoHotkey runtime, the legal docs, and notably **`data\changelog.json`** (shipped by the installer, read by the "What's New" view).

**One-time migration** (`MigrateLegacyDataIfNeeded` → `_MoveLegacyItem`), run by `BootstrapDataDir()` at every entry process's startup, before the first config read:
- Only when installed (`GetDataDir() != A_ScriptDir`) and a legacy `{app}` footprint exists.
- **Move-only, never-overwrite, non-destructive**: per item, moves only when the source exists and the target does not; `FileMove(…,0)`/`DirMove(…,1)`; any failure is caught and **leaves the source intact** (never delete-then-write). Idempotent: a re-run never re-clobbers an already-migrated (or user-edited) file.
- `changelog.json` is explicitly skipped (bundled). `audio\`/`logs\` move wholesale as directories.

**Seed** (`SeedConfigIfMissing`): if no config exists and nothing migrated, copy the bundled clean `config.example.json` template → `%APPDATA%\QuickSay\config.json` (overwrite=0). This makes the app the single authority on seed+migration and **avoids an installer/migration race** (had the installer seeded `%APPDATA%` on upgrade, migration would skip and abandon the user's real `{app}` config).

**`setup.iss`:** `[Dirs]` now creates the `%APPDATA%\QuickSay\` tree (`uninsneveruninstall`); `OnboardingAlreadyDone` checks the new marker **and** the legacy `{app}` one (so upgrades don't re-run onboarding); the uninstall keep-prompt targets `%APPDATA%\QuickSay\` and **preserves `license.dat`** (it deletes only `\data`, `config.json`, `dictionary.json`; `license.dat` sits directly under the data root, untouched).

**T2.3 / T2.1 reconciliation.** T2.3's `License_DatPath()` already hardcodes the canonical target `EnvGet("APPDATA") "\QuickSay\license.dat"`, so in **production** `GetDataDir()` and `License_DatPath()` resolve to the **same** `%APPDATA%\QuickSay\` directory — config and trial/license co-locate with **no change to `lib/license.ahk`** (the session's "reconcile only if T2.3 hardcoded a *different* path" clause did not fire). Trial state is stored **inside `license.dat`** (there is no separate `trial.dat`), so preserving `license.dat` on uninstall preserves the whole anti-abuse design.

**Files:** `lib/datadir.ahk` (new), `QuickSay.ahk` (globals, ~4 config-bypass reads, marker, 53 debug sites, `BootstrapDataDir()` in tray + `--settings` paths), `lib/settings-ui.ahk` (statics), `settings_ui.ahk` (+include +bootstrap), `onboarding_ui.ahk` (+include, config static, marker, 6 debug sites, +bootstrap), `setup.iss` ([Dirs], OnboardingAlreadyDone, uninstall prompt).
**Tests:** `tests/datadir/datadir-tests.ahk` (22 assertions — resolver, move primitive, full migration incl. changelog-skip / true-move / idempotency / non-destructive, seed) + parse probes of all three entry scripts.

### P1 #2 — T1.3-001: installer shipped the developer's live `config.json`

**Before.** `setup.iss:72` shipped `config.json` (untracked on disk, line 9 = the dev's **DPAPI-encrypted `groqApiKey`** blob, `launchAtStartup:1`, personal widget pos). Every fresh install inherited it; the blob (~240 chars, not `gsk_`) tripped onboarding's "key looks set" heuristic → `MarkOnboardingDone()` → wizard suppressed → but the blob is undecryptable on the user's machine → **transcription fails on first use with no re-prompt**. And `launchAtStartup:1` silently armed autorun on every install (feeds #4).

**After.** The installer no longer ships `config.json` **or** `dictionary.json`. It ships the pristine **`config.example.json`** as a read-only template; the app seeds `%APPDATA%\QuickSay\config.json` from it on first run (see #1). `config.example.json` was already clean (`groqApiKey:""`, `launchAtStartup:0`) — fixed only `soundTheme:"default"` → `"subtle"` (T1.4-019 landmine: `default` resolves to no theme dir). Absence of `groqApiKey` makes onboarding correctly stay armed until the user enters a real `gsk_` key.
**Files:** `setup.iss` `[Files]`, `config.example.json`. **Tests:** `tests/installer/installer-hygiene-test.ps1` (#2 section — seed clean, no DPAPI blob, `launchAtStartup=0`, real theme, no live config/dict shipped).

### P1 #4 — T1.3-025: orphaned `HKCU\…\Run\QuickSay` autorun survived uninstall

**Before.** The app (settings process) writes `HKCU\…\Run\QuickSay → "{app}\QuickSay.exe"`. The installer owned a *different* startup mechanism (`{userstartup}` shortcut) and never knew about the `Run` value → uninstall left it pointing at a deleted exe → ghost autorun error at every login.

**After.** Added `[UninstallRun]` to `reg delete` the `HKCU\…\Run\QuickSay` value on uninstall. **Deliberately NOT** app-deletes-on-exit (that would defeat the legitimate autorun feature — the value is *meant* to persist across reboots when enabled). Compounding fixed by #2's `launchAtStartup=0` default (the key is no longer armed on every fresh install).
**Files:** `setup.iss` (new `[UninstallRun]`). **Tests:** installer-hygiene `#4` section.

### P1 #3 — T1.3-011: half-done release left bumped source with no rollback

**Before.** `release.ps1` STEP 1 mutates ~7 source files + `changelog.json` + `VERSION` **before** compile/sign/ISCC/R2/version.json-signing. A mid-run failure (the documented Azure-sign hang, ISCC, R2, or a missing Ed25519 key in STEP 6) left the tree dirty with a bumped-but-unreleased version; STEP 7b commits only website files. No `try/finally`, no rollback.

**After.** New pure, dot-sourceable helper **`scripts/release-rollback.ps1`** (`Get-ReleaseSnapshotPaths` / `Save-ReleaseSnapshot` / `Restore-ReleaseSnapshot`). `release.ps1` snapshots every file the rewrite path can touch **before** STEP 1 (the list is **DERIVED from T1.6's `$VersionTargets`** so new version locations are auto-protected, plus `LICENSE_AGREEMENT.rtf`/`changelog.json`/`VERSION`), wraps STEP 1 → end in `try { … } finally { … }`, and **restores byte-for-byte on any non-success exit/throw** (gated by `$ReleaseSucceeded`, which only `-SyncOnly` and normal completion set; `-DryRun` mutates nothing). Built **on** T1.6's release.ps1 (didn't clobber `$VersionTargets`/`-CheckSync`) and **wraps** T2.5's STEP-6 signing.
**Files:** `release.ps1`, `scripts/release-rollback.ps1` (new). **Tests:** `tests/release/rollback-test.ps1` (snapshot-list derivation, mutate→restore round-trip with an untouched-bystander check, wiring assertions) + live `-CheckSync` (22 locations in sync ✅) and `-DryRun` (tree unchanged ✅).

### P1 #5 — T1.4-025: "Learn from Selection" reported success but the word was inert until reload

**Before.** `AddToDictionary` (`QuickSay.ahk`) mutated the in-memory `Dictionary` and rewrote `dictionary.json`, showed *"Added N correction(s)"*, but **never called `CompileDictionaryPattern`** → `DictCompiledPattern`/`DictReplacements` (what `ApplyDictionary` uses) stayed stale → the correction did nothing until the next `0x5555` reload or restart. The success toast lied.

**After.** `AddToDictionary` now calls `CompileDictionaryPattern()` after the successful `AtomicWriteFile` (the global `Dictionary` is already mutated, so the recompile picks up the new word) → the learned correction applies to the **very next** transcription.
**Files:** `QuickSay.ahk` (`AddToDictionary`). **Tests:** `tests/dictionary/dictionary-recompile-test.ps1` (static guard, **verified red on pre-fix HEAD, green on the fix**).
