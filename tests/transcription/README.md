# QuickSay — STT Regression Harness

Validates QuickSay's Whisper transcription accuracy and hallucination detection using
three audio buckets. Built in P0.2; expanded to the full corpus in T2.6.

---

## One-command invocation

```powershell
# Online mode (live Groq API calls + WER computation)
$env:GROQ_API_KEY = "gsk_..."
pwsh tests/transcription/run-stt-regression.ps1

# Offline-assert-only mode (corpus structure check, no network calls)
pwsh tests/transcription/run-stt-regression.ps1
```

Exit `0` = all assertions pass. Exit non-zero = at least one FAIL with a clear message.
Results are written to `tests/transcription/results/<timestamp>.json`.

## Corpus layout

```
audio/
  baseline/         8 clips with known ground-truth transcripts for WER tracking
  hallucination/    3-5 silence/noise clips that trigger Whisper hallucinations
  edge/             3 FFmpeg-generated edge cases (committed to git)
expected.json       Ground truth + per-clip expectations
```

### What's committed

- `audio/edge/` — 3 clips generated by FFmpeg (`silence.wav`, `white-noise.wav`, `sub-500ms.wav`). These are small (~64KB each) and are committed.
- `audio/baseline/` and `audio/hallucination/` — currently contain **synthetic placeholders** (FFmpeg tones/silence). These are committed as scaffolding.

### Replacing synthetic placeholders with real corpus

Real corpus requires external downloads. Run:

```powershell
pwsh tests/transcription/fetch-corpus.ps1
```

This prints step-by-step instructions for:
1. **LibriSpeech test-clean** (https://www.openslr.org/12) — 8-12 clips with known transcripts for WER
2. **sachaarbonel/whisper-hallucinations** (https://huggingface.co/datasets/sachaarbonel/whisper-hallucinations) — silence/noise clips that emit hallucinated text

T2.6 (transcription regression corpus session) will expand to 50+ hallucination clips, 20+ baseline clips, and add a Common Voice accents set.

## What the runner does

### Phase 1 — Baseline (WER)

For each `baseline/` clip:
- POSTs the WAV to `https://api.groq.com/openai/v1/audio/transcriptions` (`whisper-large-v3-turbo`)
- Computes Word Error Rate (WER) against the ground-truth transcript in `expected.json`
- Asserts WER ≤ threshold (default 15%, stored in `expected.json._werThreshold`)
- Prints per-clip WER and corpus-average WER

Synthetic placeholders are skipped with a SKIP status (WER not meaningful for sine tones).

### Phase 2 — Hallucination (capture)

For each `hallucination/` clip:
- POSTs to Groq and captures the raw output
- Does **not** re-implement `IsWhisperHallucination()` — the AHK filter is the authoritative judge
- Records the raw Groq response in the results JSON so T1.1/T2.6 can later assert filter coverage

### Phase 3 — Edge cases

- `silence.wav`, `white-noise.wav` — sent to API; raw output captured
- `sub-500ms.wav` — existence verified; **not** sent to API (QuickSay's 500ms guard prevents this in normal operation)

## Offline-assert-only mode

If `GROQ_API_KEY` is not set:
- Verifies `expected.json` is valid JSON
- Verifies all referenced audio files exist on disk
- Prints "offline-assert-only mode" and exits 0

This makes the harness safe for CI environments without an API key.

---

## WER threshold

The WER threshold is stored in `expected.json._werThreshold` (currently `0.15` as a placeholder).
Once real LibriSpeech clips are in place and a baseline measurement is taken, tighten this value
to the observed corpus-average WER + a small buffer (e.g., measured 4.2% → set threshold to 0.08).

The baseline doc (`docs/audit-campaign/findings/P0.2-baseline.md`) records the first measured WER
or notes "not measured — synthetic clips only / offline run."

## API contract

The runner mirrors `lib/http.ahk`'s `HttpPostFile()` multipart shape:
- `multipart/form-data` POST with fields: `file` (WAV bytes), `model`, `response_format=json`
- Authorization: `Bearer $env:GROQ_API_KEY`
- Response: `{"text": "..."}` (JSON)

This keeps the harness as a spec-check for the exact API shape QuickSay uses.
